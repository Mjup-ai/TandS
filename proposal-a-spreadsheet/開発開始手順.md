# 提案① スプシ版：開発開始手順

両方の開発を並行してスタートするための最小手順。

---

## 1. スプレッドシートを用意する

1. Google スプレッドシートを新規作成する。
2. 1枚目のシート名を **RAW_EMAILS** に変更する。
3. 1行目に次のヘッダーを入れる（`scripts/Config.js` の列順と一致）：
   - `id`, `message_id`, `received_at`, `from`, `to`, `subject`, `body_text`, `classification`, `processing_status`, `gmail_thread_url`

（TALENTS / PROJECTS はスクリプトの初回実行で自動作成される。）

---

## 2. Apps Script を開く

1. スプレッドシートのメニュー **拡張機能** → **Apps Script** を開く。
2. デフォルトの `Code.gs` は残してよい。以下のファイルを**追加**する：
   - **Config.js**：`scripts/Config.js` の内容をコピーして新規ファイル `Config.gs` として保存（拡張子は .gs でよい）。
   - **GmailToSheet.gs**：`scripts/GmailToSheet.js` の内容をコピー。
   - **ClassifyAndExtract.gs**：`scripts/ClassifyAndExtract.js` の内容をコピー。

---

## 3. 初回実行（手動）

1. **GmailToSheet** の関数 `fetchGmailToRawEmails` を選択して **実行**。
2. 初回は権限付与（Gmail 読み取り・スプレッドシート編集）を許可する。
3. 未読メールが RAW_EMAILS に追記されることを確認する。
4. **ClassifyAndExtract** の `processPendingEmails` を実行する。pending の行が分類され、TALENTS または PROJECTS に1行ずつ追加され、`processing_status` が `extracted` になることを確認する。

---

## 4. トリガーを設定する（任意）

- **拡張機能** → **Apps Script** → **トリガー** から：
  - `fetchGmailToRawEmails` を 5 分ごと
  - `processPendingEmails` を 10 分ごと  
  に設定すると、自動で取り込み・分類まで動く。

---

## 5. 次の実装（分類・抽出の本番化）

- `ClassifyAndExtract.js` の「分類だけモック」を、**OpenAI 等の API** に差し替える。
- プロンプトは [prompts/extract_talent.md](../../prompts/extract_talent.md) / [prompts/extract_job.md](../../prompts/extract_job.md) を参照。
- API キーは **プロジェクトのプロパティ**（ファイル → プロジェクトのプロパティ → スクリプトのプロパティ）に保存し、`PropertiesService.getScriptProperties().getProperty('OPENAI_API_KEY')` で取得する。

ここまでで「Gmail → スプシ取り込み → 分類・仮抽出」まで開発スタート完了。
