# 提案①：シート定義例（スキーマ対応）

各タブの列名・扱い。共通の [schemas](../schemas/) に合わせる。

---

## RAW_EMAILS

| 列名 | 型 | 説明 |
|------|-----|------|
| id | 自動採番 or UUID | 行の一意識別子 |
| message_id | 文字列 | Gmail Message-ID |
| received_at | 日時 | 受信日時 |
| from | 文字列 | 送信者 |
| to | 文字列 | 宛先 |
| subject | 文字列 | 件名 |
| body_text | 文字列 | 本文（または要約・先頭500字等） |
| classification | 文字列 | project / talent / mixed / other（抽出後に更新） |
| processing_status | 文字列 | pending / extracted / error |
| gmail_thread_url | 文字列 | Gmail で開くURL（任意） |

---

## TALENTS（人材台帳）

talent-extract の主要項目を列に。Evidence/Confidence はスプシでは省略可（要確認だけ列があればよい）。

| 列名 | 型 | 説明 |
|------|-----|------|
| id | 自動 | 行ID |
| source_email_id | 文字列 | RAW_EMAILS の id |
| hope_price_min | 数値 | 希望単価下限（万円） |
| hope_price_max | 数値 | 希望単価上限 |
| age | 数値 | 年齢 |
| employment_type | 文字列 | 雇用形態（正規化後） |
| nearest_station | 文字列 | 最寄駅 |
| work_location_preference | 文字列 | 勤務地希望 |
| skills | 文字列 | スキル（ID:年数, ... または JSON 文字列） |
| availability | 文字列 | 稼働（週5等） |
| start_available_date | 文字列 | 入場可能日 |
| skill_sheet_url | 文字列 | スキルシートURL |
| confidence_flags | 文字列 | 要確認項目（カンマ区切り） |
| extracted_at | 日時 | 抽出日時 |

---

## PROJECTS（案件台帳）

project-extract の主要項目を列に。

| 列名 | 型 | 説明 |
|------|-----|------|
| id | 自動 | 行ID |
| source_email_id | 文字列 | RAW_EMAILS の id |
| price_min | 数値 | 単価下限（万円） |
| price_max | 数値 | 単価上限 |
| required_skills | 文字列 | 必須スキルID（カンマ区切り） |
| optional_skills | 文字列 | 尚可スキルID |
| work_location | 文字列 | 勤務地 |
| remote_ok | 真偽 | リモート可 |
| availability | 文字列 | 稼働条件 |
| start_period | 文字列 | 入場時期 |
| duration | 文字列 | 期間 |
| conditions | 文字列 | 個人不可等（短い文言 or JSON） |
| supply_chain_info | 文字列 | 商流（一次/二次等） |
| confidence_flags | 文字列 | 要確認項目 |
| extracted_at | 日時 | 抽出日時 |

---

## MATCHES（おすすめ一覧）

| 列名 | 型 | 説明 |
|------|-----|------|
| id | 自動 | 行ID |
| project_id | 文字列 | PROJECTS の id |
| talent_id | 文字列 | TALENTS の id |
| score | 数値 | 0–100 |
| is_recommended | 真偽 | score >= 70 |
| exclusion_reason | 文字列 | Hard Filter で除外した場合の理由 |
| recommendation_reasons | 文字列 | 推薦理由（3点を改行 or カンマで） |
| attention_point | 文字列 | 注意点（1点） |
| confirmation_questions | 文字列 | 確認質問（2–3点） |
| email_draft | 文字列 | 提案メール下書き |
| call_memo | 文字列 | 電話メモ（30秒） |
| created_at | 日時 | 作成日時 |

---

## SKILL_DICT

| 列名 | 型 | 説明 |
|------|-----|------|
| skill_id | 文字列 | 正規化ID（JAVA, REACT 等） |
| name | 文字列 | 正式名 |
| aliases | 文字列 | 別名（カンマ区切り） |
| category | 文字列 | カテゴリ（任意） |
| updated_at | 日時 | 最終更新 |
| updated_by | 文字列 | 更新者 |

---

## CONFIG

| 列名 | 型 | 説明 |
|------|-----|------|
| key | 文字列 | 設定キー（score_threshold, weight_required_skill 等） |
| value | 文字列 | 値 |

マッチ計算時にここを参照する（スプシ版では読み取り専用で運用可）。

---

以上をスプシの「1行目＝ヘッダ」として定義し、Apps Script や手動入力と整合させる。
