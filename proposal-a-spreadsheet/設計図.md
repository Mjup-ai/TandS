# 提案① スプシ版：設計図

Gmail → スプシ取り込み → AI抽出 → 絞り込み・マッチ・テンプレまでをスプレッドシートで完結させる設計の一覧。

---

## 1. 全体構成

```
Gmail（集約用アカウント/ラベル）
    │
    ▼ [Apps Script: 取り込みトリガー]
RAW_EMAILS（受信一覧）
    │
    ▼ [Apps Script: 分類・抽出トリガー or 手動]
    ├─→ TALENTS（人材台帳）
    └─→ PROJECTS（案件台帳）
              │
              ▼ [マッチ計算・手動 or スクリプト]
         MATCHES（おすすめ一覧）
              │
参照用 ────────┼── SKILL_DICT（スキル辞書）
              └── CONFIG（閾値・重み）
```

---

## 2. タブ一覧と用途

| タブ名 | 用途 |
|--------|------|
| **RAW_EMAILS** | 受信メール一覧。Gmail から取り込み。分類・処理状況を更新。 |
| **TALENTS** | 人材台帳。AI 抽出結果を1行＝1人材で格納。 |
| **PROJECTS** | 案件台帳。AI 抽出結果を1行＝1案件で格納。 |
| **MATCHES** | おすすめ一覧。案件×人材のスコア・理由・注意点・確認質問・テンプレ。 |
| **SKILL_DICT** | スキル辞書。スキルID・正式名・別名・カテゴリ。 |
| **CONFIG** | 設定。スコア閾値(70)、重み、入場許容日数など。 |
| **LOGS** | 変更履歴（任意）。 |

## 4. Apps Script 処理フロー

### 4.1 取り込み（Gmail → RAW_EMAILS）

1. メール取得：`GmailApp.search('label:xxx is:unread')` 等でスレッド取得。
2. 重複チェック：Message-ID（または 件名+日付+From）が RAW_EMAILS に既にあればスキップ。
3. シート追記：RAW_EMAILS の次の空行に  
   `message_id`, `received_at`, `from`, `to`, `subject`, `body_text`, `classification`（空）, `processing_status`（pending）, `gmail_thread_url`（任意）を書き込む。

**トリガー例**: 5分ごとの時間駆動。

### 4.2 分類・抽出（RAW_EMAILS → TALENTS / PROJECTS）

1. `processing_status = pending` の行を取得（上限 N 件）。
2. 本文を AI（OpenAI 等）に送り、分類（project / talent / mixed / other）＋項目抽出。
3. 人材と判定 → TALENTS に1行追記。案件と判定 → PROJECTS に1行追記。
4. RAW_EMAILS の該当行の `processing_status` を `extracted` に更新。`classification` も更新。

**トリガー例**: 10分ごとの時間駆動、または手動「未処理メールを抽出」。

### 4.3 シートとの対応（書き込み元）

| シート | 書き込み元 | 内容 |
|--------|------------|------|
| RAW_EMAILS | メール取得スクリプト | 1メール = 1行 |
| TALENTS | 抽出スクリプト（人材と判定時） | 1行 = 1人材 |
| PROJECTS | 抽出スクリプト（案件と判定時） | 1行 = 1案件 |

---

## 5. 制限・注意

- **実行時間**: Apps Script は 6分/回の制限。件数が多い場合は「未処理の先頭 N 件だけ」で実行。
- **本文**: 長い場合は `body_text` に全文を入れず、要約 or 先頭 N 文字＋「全文は Gmail で確認」とする。
- **AI API**: API キーは Script のプロパティに保存し、コードに直書きしない。
- **原本**: 本文全文は Gmail に残す。スプシには要約 or リンクのみ。添付は「添付あり」フラグ＋ファイル名程度にする。
