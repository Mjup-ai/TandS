# 次にやること（専用アカウント2つの連携）

人材用・案件用の**各専用アカウント**があるので、その2つをシステムに連携して自動受信できるようにする。

---

## やることの順番

| 順番 | やること | 誰がやる | メモ |
|------|----------|----------|------|
| **1** | **Gmail 側の準備**（2アカウントとも Gmail の場合） | クライアント or 管理者 | Google Cloud でプロジェクト作成、Gmail API 有効化、OAuth 同意画面・認証情報（クライアントID/シークレット）を用意。2アカウント分の「メール読み取り」許可を取る。 |
| **2** | **認証情報をプロジェクトに渡す** | 開発側 | クライアントID・シークレットを環境変数 or 設定ファイルで受け取り、OAuth のリフレッシュトークンを2アカウント分取得・保存する。 |
| **3** | **バックエンドに「Gmail 取り込み」を実装** | 開発側 | 各アカウントの未読（または指定ラベル）を Gmail API で取得 → 既存の `RawEmail` に保存。アカウントが人材用/案件用なら、保存時に `classification` を `talent` / `project` で設定。 |
| **4** | **定期実行を組む** | 開発側 | 5分ごとなどで「2アカウントから未読取得 → RawEmail に保存」を実行。cron や Cloud Scheduler、またはアプリ内の setInterval など。 |
| **5** | **動作確認** | クライアント・開発 | 専用アカウントにテストメールを送り、受信一覧に自動で並ぶか確認。 |

---

## 1. Gmail 側の準備（クライアントに依頼する場合）

次のことを依頼する。

1. **Google Cloud コンソール**でプロジェクトを作成（または既存を使用）。
2. **Gmail API** を有効化。
3. **OAuth 2.0** の「認証情報」で「デスクトップアプリ」または「ウェブアプリ」タイプのクライアントID・シークレットを発行。
4. **OAuth 同意画面**で、スコープに `https://www.googleapis.com/auth/gmail.readonly`（メール読み取り）を追加。
5. **2つの専用アカウント**それぞれで、一度だけ「このアプリにメール読み取りを許可」する画面にアクセスし、同意する。→ 開発側がリフレッシュトークンを保存する。

※ 社内の Google Workspace の場合は、管理者が API 利用とアプリの許可を承認する必要がある場合あり。

---

## 2. 開発側で実装するもの（概要）

- **Gmail API 呼び出し**：未読一覧取得 → 各メールの本文・件名・From・日付を取得 → 既存の `POST /api/raw-emails` 相当の処理で `RawEmail` に 1 件ずつ保存。
- **2アカウント対応**：保存しておいた2つ分のリフレッシュトークンで、それぞれ Gmail API にアクセス。取得したメールはどちらも同じ `raw_emails` テーブルに入れ、`classification` は「アカウントA＝人材用」「アカウントB＝案件用」なら自動で `talent` / `project` を付与。
- **重複防止**：`messageId` が既に DB にあればスキップ。
- **定期実行**：上記を 5 分ごとなどで実行するジョブを用意。

---

## 3. まとめ

**次にやること**は、  
① Gmail 側の準備（API 有効化・OAuth 認証情報・2アカウントの同意）  
→ ② 認証情報を開発に渡す  
→ ③ バックエンドに Gmail 取り込み＋2アカウント対応＋定期実行を実装  
→ ④ 動作確認  

この順で進めると、各専用アカウントのメールが自動でアプリの受信一覧に並ぶようになる。

---

## 費用（Gmail API）

- **Gmail API の利用料金は無料**です（Google の公式）。
- 制限は「割り当て（クォータ）」のみ。メールの**取得**（messages.list / messages.get）は 1 回あたり数ユニットで、**2アカウントで 5 分ごとに未読を取る**程度なら、無料枠内に収まります。
- Google Cloud のプロジェクト作成も無料。課金が発生するのは、別の有料 API（例：大量の Cloud ストレージなど）を使った場合です。
- **まとめ**：専用アカウント2つのメールを取り込むだけなら、**追加費用はかからない**想定でよいです。
