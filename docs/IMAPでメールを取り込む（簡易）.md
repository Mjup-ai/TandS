# IMAP でメールを取り込む（簡易）

Gmail API ではなく **IMAP** で専用アカウントからメールを取る場合の手順。

---

## 独自ドメイン（@自社ドメイン）を使っている場合

メールアドレスが **@gmail.com ではなく @自社ドメイン**（例: `ses@company.co.jp`）でも、**どこのサーバーでメールを預かっているか**で決まる。

| 預かり先 | 取り方 |
|----------|--------|
| **Google Workspace**（独自ドメインで Gmail 運用） | **Gmail API** のまま。アドレスが @company.co.jp でも、バックエンドが Google なら Gmail API で取得。 |
| **Microsoft 365 / Exchange**（独自ドメインで Outlook 運用） | **Microsoft Graph API** または **IMAP**（有効なら）。 |
| **自社サーバー or レンタルサーバー** | **IMAP**（または POP）。管理者にホスト・ポート・認証を確認。 |

※ **独自ドメイン = アドレスの見た目**。**実体がどこで動いているか**を聞けば、Gmail API か IMAP/Graph かが決まる。

---

## 1. クライアントに聞くこと

| 聞くこと | 例 |
|----------|-----|
| **IMAP サーバー（ホスト）** | `imap.example.com`、Outlook なら `outlook.office365.com` |
| **ポート** | 通常 993（SSL） or 143（STARTTLS） |
| **アカウント（メールアドレス）** | 専用アカウント2つのアドレス |
| **パスワード** | 通常ログイン用、または **アプリ用パスワード**（2段階認証なら必須） |

※ 2アカウントなら、上記を**2セット**用意する。

---

## 2. 実装の流れ（Node の場合）

1. **ライブラリ**  
   - `imap`（[node-imap](https://www.npmjs.com/package/imap)）や **imapflow** などで IMAP 接続。
2. **接続**  
   - ホスト・ポート・user・password で接続。必要なら `tls: true`。
3. **受信トレイを開く**  
   - `INBOX` または指定フォルダを open。
4. **未読を検索**  
   - `search([['UNSEEN']])` で未読の sequence を取得。
5. **各メールを fetch**  
   - `fetch(seq, { bodies: '' })` で本文取得。ヘッダから From / Subject / Date をパース。
6. **既読にするか**  
   - 必要なら `addFlags(seq, ['\\Seen'])`。運用で「未読のまま取る」か決める。
7. **DB に保存**  
   - 取得した内容を既存の **RawEmail** に 1 件ずつ保存。`messageId` があれば重複スキップ。
8. **定期実行**  
   - 5 分ごとなどで上記を実行。2アカウントなら 2 回接続してそれぞれ実行。

---

## 3. 注意点

- **アプリパスワード**  
  Gmail でも IMAP を使う場合は、2段階認証のうえで「アプリパスワード」を発行して使う。
- **接続数・レート**  
  サーバーごとに同時接続数やレート制限があるので、連続で大量取得しすぎない。
- **文字コード**  
  本文・件名は ISO-2022-JP / UTF-8 など混在するので、デコードしてから DB に保存する（既存の .eml 取り込みと同様）。

---

## 4. このシステムとのつなぎ方

- IMAP で取ったメールは、**Gmail API のときと同じ**「1通 = RawEmail 1件」で保存する。
- あとは **分類（人材/案件）→ 台帳 → マッチ** の流れはそのまま。  
  **専用アカウントが2つ**なら、接続ごとに「このアカウントは人材用」などと紐づけて、保存時に `classification` を付けてもよい。
