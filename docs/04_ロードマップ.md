# 4. 実装ロードマップ（失敗しない順番）

## MVP1：最短で効果が出る「台帳化・探せる」

**目標**: メールを案件/人材の台帳にし、検索で「探せる」状態にする。

- メール集約 → 分類 → 抽出 → 台帳化（案件/人材）
- 検索：AND/NOT（およびOR）＋プリセット
- スキル辞書のID化（Java / JavaScript 問題の防止）
- 「要確認」運用（confidence 低の項目をフラグ）

**成果**: 初回候補提示までの時間短縮、探す時間の削減。

---

## MVP2：「動ける」にする

**目標**: マッチ理由と次のアクションまで出し、営業がそのまま動けるようにする。

- 自動マッチ：Hard Filter → Score(0–100)
- 推薦理由（3点）・注意点（1点）・確認質問（2–3点）・テンプレ文の生成
- 名寄せ：まずは**手動統合ボタン＋候補提示**（同一人物/同一案件の候補を表示し、人が確定）

**成果**: 「なぜこの候補か」の説明と、提案メール/電話メモで次の行動が明確になる。

---

## 最強版：コアラ超えの完成

**目標**: 名寄せ自動化と商流最適化で、二重提案・条件悪化を防ぎ、分析まで可能にする。

- 名寄せの自動化（類似度・添付シートハッシュ等で候補提示し、確定は人）
- 商流最適：`project_offers` / `talent_offers` を活用し、**最良条件（商流浅い・単価良い）を上に表示**
- 進捗・重複提案防止・分析（BP別品質、成約率、単価相場など）

**成果**: 同一人材の複数オファーを束ねて最良条件で提案、同一案件の複数ルートを比較して商流を最適化。

---

## マイルストーンまとめ

| フェーズ | 主な機能 | 成功の目安 |
|----------|----------|------------|
| MVP1 | Ingest〜台帳、検索、辞書、要確認 | 候補提示時間 1/10、探す時間削減 |
| MVP2 | マッチ、理由・質問・テンプレ、名寄せ手動 | 提案数増、納得感・次の行動 |
| 最強版 | 名寄せ自動、商流最適、分析・防止 | 二重提案防止、条件最適、分析活用 |

実装は上記の順で進め、各フェーズで「業務で使える状態」を一度ずつリリースすることを推奨する。
