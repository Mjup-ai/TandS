# 提案比較と開発方針（2パターン）

スプレッドシート版（①）と専用システム版（②）の2パターンを開発する。**まず①で回して効果検証 → ②で本番化**するストーリーを推奨する。

---

## 提案①：スプレッドシート版（MVP / まず回す）

### 概要

Gmail 等から取り込んだ「案件・人材」情報を AI で項目抽出し、スプレッドシートに整理。スプシ上で絞り込み・簡易マッチ・テンプレ生成まで行う。

### 主な機能

| 機能 | 内容 |
|------|------|
| 取り込み | Gmail → スプシ自動取り込み（Apps Script） |
| 分類・抽出 | 案件/人材の自動分類 ＋ 項目抽出（スキル/単価/商流/勤務地/開始時期/NG 等） |
| 検索 | フィルタ・プリセットビュー |
| マッチ | 簡易マッチ（スコア）＋ 理由/確認質問/提案テンプレ文 |

### 期間・費用相場

- **期間**: 1〜3週間
- **初期**: 30〜300万円
- **月額**: 1〜12万円/月（主に AI 利用料。添付解析が多いと上振れ）

### 向いている

- まず効果検証したい
- 少人数・短期で形にしたい
- 抽出項目・辞書・条件を現場で確定したい

### 限界

- データ量増で重くなる
- 権限/監査が弱い
- 名寄せの高度化が難しい

### 開発用フォルダ

→ **[../proposal-a-spreadsheet/](../proposal-a-spreadsheet/)** を参照。

---

## 提案②：専用システム版（Web＋DB / 本番運用）

### 概要

メール/ファイルを取り込み、DB で正規化して「検索・名寄せ・マッチ・進捗」を一気通貫で管理する専用アプリ（ブラウザで利用）。

### 主な機能（スプシ版 ＋ 強化）

| 機能 | 内容 |
|------|------|
| 取り込み | Gmail API / IMAP 等の自動化、**原本保全**（添付/URL も保存） |
| DB・検索 | **正規化 DB**：高速検索（AND/NOT/条件プリセット） |
| 名寄せ | **同一人物/同一案件の統合** ＋ 商流・条件違い（Offer）管理 |
| マッチ | 除外理由 ＋ スコア内訳 ＋ **根拠表示** |
| 進捗 | 提案〜面談〜成約のステータス、**重複提案防止ログ** |
| 運用 | 権限管理、監査ログ、バックアップ、運用ダッシュボード |

### 期間・費用相場

- **期間**: MVP 4〜8週間 / 本番 2〜4ヶ月
- **初期**: 150〜1500万円（範囲で大きく変動）
- **月額**: 4〜20万円/月（本文中心）〜 15〜50万円/月（添付多め）

### 向いている

- 毎日大量、運用人数増
- データ資産化して精度改善したい
- 本番前提（権限・監査・名寄せ）

### 開発用フォルダ・設計

→ **[../proposal-b-webapp/](../proposal-b-webapp/)** を参照。  
共通設計（スキーマ・抽出スキーマ・マッチング仕様）は **schema/** ・ **schemas/** ・ **specs/** ・ **docs/** をそのまま利用する。

---

## 推奨ストーリー：まず① → ②へ移行

1. **まず①（スプシ版）で最短導入**  
   運用にフィットする**抽出項目・辞書・条件**を確定し、効果（提案数・時間短縮）を検証する。
2. **効果が見えたら②へ移行**  
   名寄せ・権限・監査・高速化で**本番化**する。抽出スキーマ・マッチ仕様は共通なので、作り直しを最小にできる。

→ **失敗確率が一番低い**進め方。

---

## 2パターンで共通にしておくもの

両方で開発する場合、以下は**共通設計**にしておくと①→②の移行が楽になる。

| 項目 | 共通化の内容 |
|------|--------------|
| 抽出項目 | 案件/人材の**固定JSONスキーマ**（schemas/project-extract.json, talent-extract.json）を同じにする |
| マッチ仕様 | Hard Filter・スコア重み・閾値70・理由/質問/テンプレの考え方を同じにする（specs/マッチング仕様.md） |
| 辞書 | スキルID・雇用形態・駅などの**正規化ルール**を揃える（specs/正規化辞書仕様.md） |

①ではスプシの列＝このスキーマの項目、②では DB のカラム＝同じスキーマ、としておく。

---

## まとめ

|  | 提案① スプシ版 | 提案② 専用システム版 |
|---|----------------|------------------------|
| 目的 | 効果検証・短期で回す | 本番運用・データ資産化 |
| 開発 | [proposal-a-spreadsheet/](../proposal-a-spreadsheet/) | [proposal-b-webapp/](../proposal-b-webapp/) ＋ 既存 schema/docs |
| 推奨 | **まずここから** | ①で検証後に移行 |

この2パターンを開発する場合、**①の実装と②の設計を並行**してもよいし、**①をリリースしてから②**に着手してもよい。
