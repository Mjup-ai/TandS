# スキル辞書の運用（誰が更新するか）

辞書が育たないと「Java と JavaScript が同じ」問題が再発する。**誰が・どこで・どう更新し、ログをどう残すか**を決める。

---

## 1. 方針

- スキルは **ID 化必須**。検索・マッチはスキル ID で行う。
- 新規表記（同義語・略称）は **運用で辞書に追加**していく。更新ログを必ず残す。

---

## 2. 誰が更新するか

| フェーズ | 担当 | 更新場所 |
|----------|------|----------|
| MVP（管理画面なし） | 運用担当・開発 | **SKILL_DICT シート**で更新。スキルID・正式名・別名（カンマ区切り）・カテゴリを編集。 |
| 管理画面あり | 運用担当 | 管理画面の「スキル辞書」で追加・編集。 |

**ルール**: 辞書の変更は **「誰がいつ何を変えたか」** を必ず記録する（更新ログ必須）。

---

## 3. SKILL_DICT シート（MVP 時）

スプレッドシート完結版では、次の列を用意する。

| 列 | 説明 |
|----|------|
| スキルID | 正規化コード（例: JAVA, JAVASCRIPT, REACT） |
| 正式名 | 表示用（例: Java） |
| 別名 | カンマ区切り（例: Java,Java8,java） |
| カテゴリ | 言語/フレームワーク/インフラ等（任意） |
| 最終更新日 | 手動 or 更新時に記録 |
| 更新者 | 誰が変更したか |

**インポート**: シートを CSV/JSON でエクスポートし、バッチで DB の `skills` / `skill_aliases` に反映する、またはスheet を「正」として参照する実装でも可。

---

## 4. 更新ログ（必須）

- テーブル **skill_update_log** を用意する（推奨）。  
  項目: `id`, `skill_id`, `action`(add/update/delete/alias_add), `old_value`, `new_value`, `updated_at`, `updated_by`  
  または、`activities` の `target_type: skill`, `action: update` で代用する。
- 管理画面がない場合は、**SKILL_DICT シートの「更新履歴」タブ**に「日付・変更内容・担当」を手動で残す運用でも可。ただし「ログ必須」は守る。

---

## 5. 未登録スキルの扱い

- 抽出結果に辞書にないスキル表記が出た場合: **「未登録」としてフラグを立て、一覧で確認できるようにする**。
- 運用で正式名・別名を決めて辞書に追加したら、該当データを再正規化（バッチ or 手動トリガー）する。

これで「辞書が育つ」仕組みと「Java 問題」の再発防止を両立する。
