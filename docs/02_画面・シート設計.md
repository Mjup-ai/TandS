# 2. 画面/シート設計

## スプレッドシート完結版（MVP）タブ構成

※ 裏側はDBで保持し、シートは「ビュー」として利用。後でWebアプリに移行しても作り直しを防ぐ。

### タブ一覧

| タブ名 | 用途 | 主な項目（例） |
|--------|------|----------------|
| **RAW_EMAILS** | 受信一覧 | Message-ID, 受信日時, From, Subject, 分類, 処理状況, 原文リンク |
| **TALENTS** | 人材台帳 | 名寄せID, 希望単価, 年齢, 雇用形態, 最寄駅, スキル(ID), 稼働, 入場可能日, スキルシート参照, 最終更新 |
| **PROJECTS** | 案件台帳 | 名寄せID, 単価帯, 必須/尚可スキル, 勤務地, リモート可否, 稼働, 入場時期, 条件, 商流, 最終更新 |
| **MATCHES** | おすすめ一覧 | 案件, 人材, スコア, 内訳, 推薦理由(3), 注意点(1), 確認質問(2–3), 除外理由(Hard Filter時) |
| **SKILL_DICT** | スキル辞書 | スキルID, 正式名, 別名(カンマ区切り), カテゴリ |
| **CONFIG** | 設定 | スコア閾値(70), 各項目の重み, 入場許容日数, その他パラメータ |
| **LOGS** | 変更履歴 | 日時, 操作種別, 対象ID, ユーザー, 備考 |

## 各タブの項目詳細（開発用）

### RAW_EMAILS

- `id`, `message_id`, `received_at`, `from`, `to`, `subject`
- `body_text`, `body_html`（またはストレージ参照）
- `classification` (project | talent | mixed | other)
- `processing_status` (pending | extracted | normalized | resolved | error)
- `raw_attachments_count`, `raw_links_count`
- `created_at`, `updated_at`

### TALENTS（人材台帳ビュー）

- `talent_id`（名寄せ後の本体ID）, `offer_id`（オファー単位で見る場合は別）
- `hope_price_min`, `hope_price_max`, `age`, `employment_type_id`
- `nearest_station_id`, `work_location_preference`
- `skills`（スキルID + 経験年数 + 直近使用の構造）
- `availability`（週5等）, `start_available_date`
- `skill_sheet_url`, `skill_sheet_attachment_id`
- `confidence_flags`（要確認項目）, `source_email_id`
- `created_at`, `updated_at`

### PROJECTS（案件台帳ビュー）

- `project_id`, `offer_id`（商流別の場合は別）
- `price_min`, `price_max`
- `required_skill_ids`, `optional_skill_ids`
- `work_location`, `remote_ok`, `availability`（稼働条件）
- `start_period`, `duration`, `conditions`（個人不可・年齢等）
- `supply_chain_info`（一次/二次・BP名等）
- `confidence_flags`, `source_email_id`
- `created_at`, `updated_at`

### MATCHES

- `project_offer_id`, `talent_offer_id`, `project_id`, `talent_id`
- `score` (0–100), `score_breakdown` (JSON)
- `is_recommended` (score >= 70)
- `hard_filter_failed` (boolean), `exclusion_reason`
- `recommendation_reasons` (3点), `attention_point` (1点)
- `confirmation_questions` (2–3点)
- `email_draft`, `call_memo`（30秒用）
- `created_at`

## Webアプリ化時の画面イメージ（将来）

- **ダッシュボード**: 今日の新着、プリセット検索、直近マッチ
- **案件一覧 / 人材一覧**: ファセット検索、保存条件、並び替え
- **案件詳細 / 人材詳細**: 抽出値・根拠行・confidence・要確認
- **マッチ結果**: スコア・内訳・理由・注意点・質問・テンプレ
- **名寄せ管理**: 同一候補の統合・手動マージ
- **設定**: CONFIG に相当する閾値・重み・辞書の編集

以上を「開発に渡せる粒度」の画面/シート設計とする。
