# 完璧に使える状態 → そのあと API 連携

このシステムを**まず完璧に使える**まで進化させ、**そのあと** Gmail API 等の外部連携を足す。

---

## 「完璧に使える」の定義

| 項目 | 内容 |
|------|------|
| **受信** | 手動で1件追加できる。件名 or 本文必須のバリデーション。一覧・ページネーション・分類（案件/人材）が迷わずできる。 |
| **案件** | 案件一覧が表示され、詳細（本文全文）を確認できる。 |
| **人材** | 人材一覧が表示され、詳細を確認できる。 |
| **マッチ** | 案件×人材の候補が表示される。スコア・推薦閾値（70点）・除外理由・スコア内訳が分かる。推薦だけ絞り込み可能。 |
| **全体** | ヘッダーで件数サマリ（受信N / 案件N / 人材N）が分かる。エラー時はメッセージで分かる。設定（閾値）を確認できる。 |

→ ここまでを「完璧に使える」とする。**入力は手動で十分**（コピペでメールを入れる）。

---

## API 連携は「そのあと」のフェーズ

- **Gmail API / IMAP** で「届いたメールを自動取り込み」するのは、上記が整ってから追加する。
- 理由：システムの操作・マッチ・設定が固まっていないうちに自動取り込みを入れると、不具合の切り分けが難しくなる。
- 順番：**完璧に使える（手動入力で全フローが回る）→ 運用で確認 → そのあと Gmail API 等を接続**。

---

## 実装済みで「完璧」に近づけているもの

- **件数サマリ**：ヘッダーの受信・案件・人材の件数表示
- **マッチ**：スコア内訳（基礎＋キーワード）、推薦フィルタ（推薦だけ表示）、閾値の説明
- **案件・人材**：一覧で行をクリックすると本文全文を表示（開閉）
- **受信**：件名・本文どちらか必須のクライアント側バリデーション
- **設定**タブ：推薦閾値（70点）の確認

API 連携の設計・手順はルートの [docs/取り込み仕様.md](../../docs/取り込み仕様.md) を参照。
