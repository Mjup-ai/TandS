# 届くメールのサンプル（案件・人材）

※ サンプルは「こんなメールが届く」という**共有用**。本番では **Gmail API 等で連携し、アプリ側で自動受信**する想定。

実際に届くメールの内容は、以下のサンプルフォルダで確認できる。

- **案件**: `/Users/mjup/Downloads/SES営業サンプルメール（案件）/`
- **人材**: `/Users/mjup/Downloads/SES営業サンプルメール（人材）/`

いずれも **.eml** 形式。件名・差出人・日付・本文が含まれる。

---

## フォーマットの特徴

- **文字コード**: ISO-2022-JP / UTF-8 が混在。Subject や本文が `=?UTF-8?B?...?=` や `=?ISO-2022-JP?B?...?=` でエンコードされていることがある。
- **本文**: `Content-Transfer-Encoding: base64` や `8bit` など。複数パート（text/plain + text/html）のメールもある。
- **案件メールの例（⑱）**: 案件概要・スキル・稼働場所・単金・清算・期間・面談・外国籍 などの見出しで整理されている。
- **人材メールの例**: 要員提案、スキル・単価・勤務形態・入場時期・スキルシートURL など。

---

## システムでの扱い

- **取り込み**: .eml をパースして From / Subject / Date / 本文（text）を取得し、`RawEmail` に保存する。
- **分類**: 件名・本文から「案件」か「人材」を判定（キーワード or AI）。サンプルはフォルダ分けで正解ラベルが分かるので、抽出・分類の精度検証に使える。
- **抽出**: 本文から単価・スキル・入場時期などを取り出す際のプロンプトや辞書は、これらのサンプルに合わせて調整する。

このシステムの「メールを1件追加」や「.eml 取り込み」で、上記サンプルをそのまま投入して動作確認できる。
