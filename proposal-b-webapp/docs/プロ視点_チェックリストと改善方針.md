# プロ視点：チェックリストと改善方針

さまざまなプロ視点で**修正案・改善案**を出し、実装に反映するための基準。

---

## 視点一覧

| 視点 | チェック内容 | 改善の方向 |
|------|--------------|------------|
| **UX** | 迷わないか、フィードバックはあるか、空状態・エラー表示は適切か | ナビ・ラベル・トースト・空状態メッセージ |
| **運用** | ログ・監査・権限・復旧 | リクエストログ、エラー時のメッセージ、将来の権限の土台 |
| **セキュリティ** | 入力検証、インジェクション、PII | バリデーション、長さ制限、サニタイズ |
| **パフォーマンス** | 件数増での応答、N+1、ページネーション | ページネーション、select 最小化 |
| **保守性** | 型・エラーハンドリング・構成 | 共通エラー形式、ルート分割、定数化 |
| **SES業務** | 名寄せ・商流・マッチング・説明可能性 | 分類→台帳→マッチの流れを画面で見せる |
| **開発体験** | 一貫した規約、ドキュメント | 修正・改善案をこの doc に追記し、実装で消化 |

---

## 修正案・改善案の運用

- **修正案**：バグ・仕様ズレの修正。優先度高。
- **改善案**：UX・性能・保守性の向上。優先度は都度つける。
- 採用した案は「実装済み」とし、日付と内容をメモする。

---

## 実装済みメモ（抜粋）

- **API**：入力バリデーション（件名 or 本文必須、長さ制限）、共通エラー形式（code, message）、受信一覧のページネーション（limit, offset）、分類更新 PATCH
- **UX**：ナビ（受信 | 案件 | 人材 | マッチ）、成功/エラーメッセージ表示、空状態の明確化、一覧の前へ/次へ
- **SES業務**：受信一覧で「人材」「案件」ボタンで分類し、台帳への振り分けの入口を表示。案件・人材・マッチタブは未実装の案内のみ。

---

## 実装済み（続き）

- **分類時に Offer 自動作成**：受信で「案件」→ Project + ProjectOffer、「人材」→ Talent + TalentOffer を自動作成
- **案件一覧**：GET /api/project-offers、案件タブで一覧表示
- **人材一覧**：GET /api/talent-offers、人材タブで一覧表示
- **マッチ一覧**：GET /api/matches（案件×人材の組み合わせ、簡易スコア70＝推薦）、マッチタブで表示
