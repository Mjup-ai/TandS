# 提案② 専用システム版：開発開始手順

両方の開発を並行してスタートするための最小手順。

---

## 1. バックエンドを動かす

```bash
cd proposal-b-webapp/backend
cp .env.example .env
npm install
npx prisma migrate dev --name init
npm run dev
```

- `http://localhost:4000/api/health` にアクセスして `{"ok":true,"service":"ses-match-backend"}` が返れば OK。
- DB は `backend/prisma/dev.db`（SQLite）に作成される。

---

## 2. フロントエンドを動かす

別ターミナルで：

```bash
cd proposal-b-webapp/frontend
npm install
npm run dev
```

- ブラウザで `http://localhost:3000` を開く。
- 画面上で「GET /api/health」の結果が表示されれば、プロキシ経由でバックエンドと繋がっている。

---

## 3. 次の実装

- **取り込み**: `.eml` アップロード or Gmail API で `RawEmail` に保存する API を追加。
- **分類・抽出**: 固定スキーマ（`schemas/`）＋プロンプト（`prompts/`）で AI を呼び、`ProjectOffer` / `TalentOffer` を作成。
- **検索**: 案件・人材の一覧 API（AND/NOT 条件）。
- **マッチ**: Hard Filter → スコア計算 → 一覧 API。

設計は [MVPスコープ.md](MVPスコープ.md) とルートの [docs/](../docs/) を参照。
