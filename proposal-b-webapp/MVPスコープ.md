# 提案②：MVP スコープ（4〜8週間）

専用システム版の MVP で「含める／含めない」を明示する。

---

## 含める（MVP で実装する）

| 項目 | 内容 |
|------|------|
| **取り込み** | 手動 .eml 投入、または Gmail API で「未読→処理」のいずれか。本文・メタは DB に保存。 |
| **分類** | 案件 / 人材 / 混在 / その他（AI またはルール）。 |
| **抽出** | 固定スキーマ（project-extract, talent-extract）で AI 抽出。根拠行・confidence を保存。 |
| **正規化** | スキルID・駅・雇用形態の辞書参照。未登録はフラグのみ。 |
| **台帳** | projects / talents / project_offers / talent_offers に格納。名寄せは「1 offer = 1 行」でよい（同一統合は候補提示まで）。 |
| **検索** | AND/NOT（およびOR）、主要条件での絞り込み。プリセット（今日の新着、単価帯等）を数本。 |
| **マッチ** | Hard Filter → スコア(0–100) → 70以上を推薦。理由・注意点・確認質問・テンプレ文を生成。 |
| **名寄せ** | 「同一っぽい候補」を一覧表示。人が「同一」を確定して既存 Talent/Project に Offer を紐づけ。 |
| **画面** | 案件一覧・人材一覧・マッチ結果・名寄せ候補の確認。設定（CONFIG 相当）は簡易（シート or 1画面）。 |
| **権限** | 閲覧者 / 編集者の2ロール程度。PII は閲覧者にマスキング or 制限。 |
| **ログ** | 取り込み・抽出・名寄せ確定は activities に記録。 |

---

## 含めない（本番で追加）

| 項目 | 備考 |
|------|------|
| **添付の実体保管** | MVP では「添付あり」＋ファイル名のみ。実体は Gmail 等に残す。本番で S3 等に保存。 |
| **完全自動名寄せ** | 候補提示＋人が確定まで。自動マージは本番で検討。 |
| **進捗・パイプライン** | 提案→面談→成約のステータス管理は本番で。MVP ではマッチ結果の「採用/見送り」のみ（評価ループ用）。 |
| **細かい権限・監査** | ロールは2種。監査ログの検索・エクスポートは本番で。 |
| **運用ダッシュボード** | 件数・成約率・BP別品質等は本番で。 |
| **自然文検索** | 「来月入場、React 3年…」→ 条件分解は本番で。MVP はファセットのみ。 |

---

## 成果の目安（MVP 完了時）

- 新着メール（.eml または Gmail）を投入すると、案件/人材に分類され、台帳に抽出結果が並ぶ。
- 条件で絞り込み、マッチ一覧でスコア・理由・確認質問・テンプレを確認できる。
- 同一人物・同一案件の候補が一覧に出て、人が確定するとオファーが統合される。
- 採用/見送りを記録し、後で重み調整に使える状態になっている。

この範囲で 4〜8週間を目安にスコープを固定する。
