# 提案②：次にやること

起動したあと、**どこから手を付けるか**の順番。

---

## 今すぐできること（確認だけ）

1. ブラウザで **http://localhost:3000** を開く。
2. 画面上に「GET /api/health」の結果（`ok: true`）が出ていれば、フロントとバックエンドは繋がっている。
3. この段階では「メールを取り込む」「検索する」画面はまだないので、**次に機能を足していく**。

---

## このあとやる順番（おすすめ）

| 順番 | やること | やったらできること |
|------|----------|---------------------|
| **1** | **メールを取り込む API を1本作る** | .eml ファイルをアップロード（または本文を POST）すると、`RawEmail` に1件保存される。 |
| **2** | **「受信一覧」画面を1枚作る** | 保存したメールが一覧で見える。 |
| **3** | **分類・抽出** | 1件選んで「分類する」を押すと、AI で案件/人材判定し、項目を抽出して `ProjectOffer` または `TalentOffer` に保存。 |
| **4** | **案件一覧・人材一覧** | 台帳を一覧表示。条件で絞り込み（AND/NOT）を追加。 |
| **5** | **マッチ** | 案件×人材のスコア計算（Hard Filter → 0–100）。一覧で「おすすめ」が並ぶ。 |
| **6** | **名寄せ候補** | 同一っぽい人材・案件を候補表示し、人が「同一」を確定。 |

まず **1 → 2** までやると「メールを入れたら一覧に出る」ができて、イメージがつかみやすい。
