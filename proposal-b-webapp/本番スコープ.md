# 提案②：本番スコープ（2〜4ヶ月）

MVP の後に本番化する際に追加する機能・非機能。

---

## 追加する機能

| 項目 | 内容 |
|------|------|
| **取り込みの安定化** | Gmail API / IMAP の定期実行、エラーリトライ、未処理キュー管理。 |
| **原本保全の完成** | 添付の実体を S3 相当 or Drive に保存。`raw_attachments.storage_key` と content_hash。保存期間は [データ保持と権限](../docs/データ保持と権限.md) に従う。 |
| **名寄せの強化** | 添付ハッシュ一致を強い証拠にした候補ランキング。同一判定キーのチューニング。自動マージはオプション（人が承認するフロー可）。 |
| **進捗・パイプライン** | 提案→面談→オファー→成約のステータス。重複提案防止（同一人材を別案件に提案したログのチェック）。 |
| **権限の細分化** | 閲覧 / 編集 / 管理者に加え、 export や名寄せ確定を分離する等。 |
| **監査・ログ** | 誰がいつ何をしたかの検索・エクスポート。activities の活用。 |
| **運用ダッシュボード** | 日別・月別の取り込み件数、成約率、BP別品質、単価相場等。 |
| **評価ループの活用** | 採用/見送り理由の集計レポート。重み・閾値の調整画面。 |
| **自然文検索** | 「来月入場、React 3年、80万まで」等を条件に分解して検索。 |

---

## 非機能・運用

| 項目 | 内容 |
|------|------|
| **バックアップ** | DB と添付ストレージの定期バックアップ。リストア手順の整備。 |
| **可用性** | 本番環境の SLA に応じた冗長化・監視。 |
| **コスト管理** | AI 利用の日/月上限、超過時のアラート・保留（[運用ルール](../docs/03_運用ルール.md)）。 |
| **セキュリティ** | PII の暗号化 at rest、アクセスログ。 |

---

## スケジュール目安

- MVP 完了後、上記を「本番スコープ」として優先度をつけて 2〜4ヶ月で段階リリースする。
- まず「取り込み安定化・原本保全・進捗」を先行し、その後に「名寄せ強化・ダッシュボード・自然文検索」を入れる進め方が現実的。
